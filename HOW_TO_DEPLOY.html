Tips & Troubleshooting
üí°
How to update the app after changes
Just upload the new app.py to GitHub (replace the old file) ‚Äî Streamlit Cloud automatically detects the change and redeploys within seconds. No manual re-deploying needed.
‚ö†Ô∏è
Important: Database resets on restart
The app uses a local SQLite database (readwithwaleed.db). On Streamlit Community Cloud, this database resets whenever the app restarts or goes to sleep (after inactivity).
For a permanent production database that survives restarts, consider upgrading to Supabase (free PostgreSQL) or PlanetScale. Ask me and I can upgrade the app to use one of these!
üîí
Keeping the app awake
Free Streamlit Cloud apps go to sleep after ~7 days of inactivity. When someone visits, it wakes up in about 30 seconds. To keep it always awake, you can use a free service like UptimeRobot to ping your URL every 5 minutes.

If deployment fails
Click "Manage app" ‚Üí "Logs" in Streamlit Cloud to see the error. Common fixes:

  ‚Ä¢ Make sure requirements.txt is in the root of your repo (not inside a folder)
  ‚Ä¢ Make sure the main file is named exactly app.py
  ‚Ä¢ Check that Python version is 3.9+
